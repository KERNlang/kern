# Agon Gap Closure Backlog

**Date:** 2026-04-16
**Status:** Proposed
**Scope:** Rebaseline Agon on current Kern, close the remaining real compiler blocker, then reduce handler-heavy escape hatches in the Agon CLI.

## Context

Agon already uses large parts of modern KERN successfully:

- `service`, `implements`, and `singleton` are in active use.
- `union` is in active use.
- default params are supported.
- `signal` and `cleanup` are supported.
- Ink screens, refs, animations, and memoized surfaces are supported in current Kern.

Two things were found during validation:

1. Agon's checked-in generated files are still stamped `@generated by kern v2.0.0`, while the local `kern-lang` workspace is `3.2.3`.
2. The old Agon planning doc overstates missing language features. The remaining real blocker is narrower: `machine` transitions still cannot express typed payload parameters or first-class guards.

`.kern-gaps` is also not a blocker list. In current Kern it records every non-empty `handler`, so it is best treated as a heat map for where KERN still relies on raw TypeScript escape hatches.

## Current Assessment

### Real compiler blocker

`machine` transitions are currently limited to `transition(entity) => entity` shapes.

This blocks Agon's `agent-state.kern`, which needs:

- typed event payloads on transitions
- guard predicates for budget/deadline/state checks

### Stale Agon gap notes

These should be revalidated and likely removed after recompile:

- `packages/cli/src/kern/surfaces/status.kern`
- `packages/cli/src/kern/surfaces/agent.kern`

### Handler heat map

Agon's remaining KERN escape-hatch volume is concentrated in the CLI package, especially:

- `packages/cli/src/kern/signals/dispatch.kern`
- `packages/cli/src/kern/cesar/brain.kern`
- `packages/cli/src/kern/surfaces/app.kern`
- `packages/cli/src/kern/blocks/engine.kern`
- `packages/cli/src/kern/cesar/session.kern`

Those are phase-2 abstraction targets, not phase-1 blockers.

## Backlog

### P0. Rebaseline Agon on current Kern

**Goal:** make Agon's generated output and gap notes reflect the current compiler, not the old `v2.0.0` state.

**Files**

- `packages/cli/src/commands/transpile.ts`
- Agon: `packages/*/src/generated/**`
- Agon: `packages/cli/src/kern/surfaces/status.kern`
- Agon: `packages/cli/src/kern/surfaces/agent.kern`
- Agon: `docs/plans/kern-compiler-gaps.md`

**Tasks**

- Rebuild `kern-lang` so the installed `kern` binary matches source.
- Recompile Agon from `.kern` using the rebuilt compiler.
- Remove stale Agon `KERN-GAP` comments that no longer reproduce on current Kern.
- Rewrite the Agon gap doc so it only lists still-open compiler blockers.
- Keep `.kern-gaps` as a handler-density signal, not as a proxy for broken features.

**Acceptance**

- Agon generated files are stamped with the current Kern version.
- `status.kern` and `agent.kern` no longer claim bugs that current output disproves.
- `kern gaps --root=<Agon>` is interpreted as an audit, not as the source of truth for compiler breakage.

### P1. Add payloaded machine transitions

**Goal:** allow machine transitions to accept typed event payloads.

**Recommended syntax**

```kern
machine name=Plan
  state name=draft
  state name=approved
  transition name=approve from=draft to=approved params="actor:string,at:number"
    handler <<<
      return { ...entity, state: 'approved' as PlanState, approvedBy: actor, approvedAt: at };
    >>>
```

**Files**

- `packages/core/src/node-props.ts`
- `packages/core/src/schema.ts`
- `packages/core/src/codegen/machines.ts`
- `packages/core/tests/codegen-core.test.ts`
- `packages/core/tests/golden-codegen.test.ts`
- `packages/core/tests/schema-validation.test.ts`

**Tasks**

- Add `params?: string` to transition props.
- Teach schema validation that transitions may declare `params`.
- Reuse the existing param parser for transition signature emission.
- Keep zero-param transitions backward compatible.
- Decide and document the payload ABI for reducer generation:
  Prefer `{ type: 'approve', payload: { actor, at } }` over string-only actions.

**Acceptance**

- Core codegen emits callable transition helpers with typed payload parameters.
- Existing machine fixtures still pass unchanged.
- New fixtures compile both zero-param and payloaded transitions.

### P2. Add machine guards

**Goal:** let a transition declare its precondition at the IR level instead of burying it inside imperative handler code.

**Recommended syntax**

```kern
transition name=start from=approved to=running params="now:number" guard="hasBudget(entity, now)"
```

**Files**

- `packages/core/src/node-props.ts`
- `packages/core/src/schema.ts`
- `packages/core/src/codegen/machines.ts`
- `packages/core/tests/codegen-core.test.ts`
- `packages/core/tests/golden-codegen.test.ts`

**Tasks**

- Add `guard` as an allowed transition prop.
- Emit guard checks before the transition body.
- Introduce a typed guard failure error instead of forcing every guard into ad hoc `if (...) throw` blocks.
- Document evaluation order:
  state validation first, guard second, handler body last.

**Acceptance**

- Guarded transitions compile to explicit runtime checks.
- Guard failures are distinguishable from invalid-state failures.
- Unguarded transitions remain unchanged.

### P3. Align machine reducer output with payloaded transitions

**Goal:** keep the Ink/terminal reducer path usable once transitions carry payloads.

**Files**

- `packages/core/src/codegen/machines.ts`
- `packages/terminal/src/transpiler-ink.ts`
- `packages/core/tests/codegen-core.test.ts`
- `packages/terminal/tests/*.ts`

**Tasks**

- Replace string-only reducer actions with typed action objects when a machine uses payloaded transitions.
- Preserve the current lightweight string-action output for zero-payload machines if that keeps generated code simpler.
- Verify the Ink transpiler path that calls `generateMachineReducer` still emits valid React reducer code.

**Acceptance**

- A machine with payloaded transitions compiles for Ink without hand edits.
- Existing simple machines remain source-compatible.

### P4. Migrate Agon `agent-state.kern` back to a real machine

**Goal:** remove the largest remaining “language workaround” that is explicitly documented in Agon source.

**Files**

- Agon: `packages/core/src/kern/cesar/agent-state.kern`
- Agon: `packages/core/src/generated/**/agent-state*.ts`
- Agon tests covering agent lifecycle transitions

**Tasks**

- Replace the union+reducer workaround with a true machine once payloads and guards land.
- Model approval requests, cancellations, failures, and budget exits as transition payloads plus guards.
- Delete the now-obsolete `KERN-GAP: machine` comments from the file header.

**Acceptance**

- `agent-state.kern` no longer documents missing machine features as blockers.
- Generated code preserves current runtime behavior.
- Agent lifecycle tests still pass.

### P5. Reduce handler-heavy orchestration hotspots

**Goal:** turn the current handler heat map into a second wave of KERN primitives, after the machine blocker is closed.

**Candidate abstractions**

- streamed async loop primitive for repeated `for await (const chunk of gen)` patterns
- polling/heartbeat primitive for repeated `setInterval` + cleanup flows
- cancellable job orchestration primitive for repeated `AbortController` setup + teardown

**Primary Agon heat-map targets**

- `packages/cli/src/kern/signals/dispatch.kern`
- `packages/cli/src/kern/cesar/brain.kern`
- `packages/cli/src/kern/surfaces/app.kern`
- `packages/cli/src/kern/cesar/session.kern`

**Files likely involved in Kern**

- `packages/core/src/schema.ts`
- `packages/core/src/node-props.ts`
- `packages/core/src/codegen/functions.ts`
- `packages/terminal/src/transpiler-ink.ts`
- `packages/core/tests/*`

**Acceptance**

- New primitives replace repeated orchestration handler patterns in at least one Agon hotspot.
- Handler character count drops materially in the migrated files.
- No regression in generated Ink behavior.

## Recommended Execution Order

1. Complete P0 so Agon's current state is truthful.
2. Land P1 payloaded transitions.
3. Land P2 guards.
4. Land P3 reducer alignment.
5. Use those features to complete P4 on `agent-state.kern`.
6. Treat P5 as follow-up optimization work, not a blocker for Agon correctness.

## Non-Goals

- Do not treat every `handler` as a bug.
- Do not design a full new workflow DSL before payloaded machines land.
- Do not add broad class/object syntax work; Agon already proves `service` is sufficient for the previously claimed class gap.
